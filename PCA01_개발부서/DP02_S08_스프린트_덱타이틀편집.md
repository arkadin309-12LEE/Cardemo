# Sprint 8 (8주차): 덱 타이틀 커스터마이징

> **이번 주 목표**: 덱 꾸미기의 핵심 기능입니다. 사용자가 덱 커버 이미지 위에 제목을 자유롭게 배치하고 꾸밀 수 있게 합니다.
> 
> **사용자 테스트 목표**: 덱 커버 이미지 위 원하는 곳에, 원하는 크기와 각도로 덱 제목을 멋지게 배치할 수 있다.

---

## 상세 작업 목록

- [ ] **(사전 작업) 기술 검증 프로토타입 제작** (총 4시간)
    1. [ ] **(목표)**: 본 기능 개발에 앞서, 제스처를 이용한 위젯 변형(이동, 확대/축소, 회전)의 기술적 구현 가능성과 복잡도를 미리 파악합니다.
    2. [ ] **(구현)**: 별도의 테스트용 `dart` 파일에서, `matrix_gesture_detector` 패키지를 사용하여 간단한 `Container` 위젯을 조작하는 예제를 만듭니다.
    3. [ ] **(검증)**: 이 프로토타입을 통해 제스처 처리의 성능이나 구현 시의 어려운 점은 없는지 미리 확인하고 기록합니다.

- [ ] **타이틀 편집 UI 구현** (총 10시간)
    - **(참고)**: 위의 프로토타입 제작 결과에 따라 예상 소요시간이 변경될 수 있습니다.
    1. [ ] **(패키지 학습)**: `matrix_gesture_detector` 패키지의 예제를 실행해보며 드래그, 스케일, 회전 값을 얻는 방법을 학습합니다.
    2. [ ] **(UI)**: 덱 커버 이미지 위에 `MatrixGestureDetector` 위젯을 배치하고, 그 자식(child)으로 `Text` 위젯(덱 타이틀)을 넣습니다.
    3. [ ] **(로직)**: 제스처가 발생할 때마다 `MatrixGestureDetector`가 반환하는 매트릭스(Matrix) 값을 `ValueNotifier`나 `StateProvider`에 저장합니다.
    4. [ ] **(UI 연결)**: `Transform` 위젯을 사용하여 `Text` 위젯에 위에서 저장한 매트릭스 값을 적용, 실시간으로 타이틀이 움직이고, 커지고, 회전하게 만듭니다.

- [ ] **타이틀 위치, 크기, 회전 값 저장 및 적용** (총 8시간)
    1. [ ] **(DB 수정)**: `decks` 테이블에 타이틀의 변환 정보를 저장할 컬럼들을 추가합니다. (예: `title_matrix` TEXT 또는 `title_x`, `title_y`, `title_scale`, `title_rotation` REAL)
    2. [ ] **(로직)**: ‘저장’ 버튼을 누르면, 현재 적용된 매트릭스 값을 문자열이나 각 속성값으로 변환하여 DB에 `UPDATE` 합니다.
    3. [ ] **(로직)**: 덱 보기 화면을 로드할 때, DB에 저장된 타이틀 변환 값을 읽어옵니다.
    4. [ ] **(UI 적용)**: 읽어온 값으로 `Matrix4`를 다시 계산하고, `Transform` 위젯에 적용하여 저장된 타이틀의 위치, 크기, 회전을 복원합니다.

- [ ] **타이틀 스타일(폰트, 색상) 변경 기능** (총 4시간)
    1. [ ] **(DB 수정)**: `decks` 테이블에 `title_font`, `title_color` 같은 컬럼을 추가합니다.
    2. [ ] **(UI)**: 타이틀 편집 모드에서 화면 하단에 색상 팔레트와 폰트 목록을 보여주는 UI를 추가합니다.
    3. [ ] **(로직)**: 사용자가 색상이나 폰트를 선택하면, 해당 값을 `StateProvider`에 업데이트하여 타이틀에 실시간으로 반영하고, 저장 시 DB에도 함께 저장합니다.
