# 카데모(CARDEMO) Phase 1 상세 작업계획서 (초심자용)

## 1. Phase 1 목표

이 단계의 목표는 **가장 기본적인 카드 메모 기능을 동작하게 만드는 것**입니다. 사용자가 앱을 처음 켰을 때 메모를 작성하고, 저장하고, 목록에서 다시 확인하고, 삭제하는 핵심적인 경험을 완성합니다. 이 단계를 마치면 앱의 뼈대가 완성됩니다.

---

## **Sprint 1 (1주차): 프로젝트 기반 구축**

> **이번 주 목표**: 코딩을 시작할 수 있는 깨끗한 도화지를 준비합니다. Flutter를 설치하고, 데이터베이스 파일을 만들 준비를 합니다.

- [x] **Flutter 개발 환경 구축** ✅ 2025-09-20
    - **(힌트)**: Flutter 공식 홈페이지의 가이드에 따라 SDK를 설치하고, `flutter doctor` 명령어로 내 컴퓨터에 필요한 모든 설정이 완료되었는지 확인하세요. 안드로이드 스튜디오나 VS Code 설치도 필요합니다.
    - **(예상 소요시간)**: 4시간

- [x] **신규 Flutter 프로젝트 생성** ✅ 2025-09-20
    - **(힌트)**: 터미널(명령 프롬프트)에서 `flutter create cardemo` 명령어를 실행하여 'cardemo'라는 이름의 프로젝트를 생성합니다. 생성된 폴더 구조를 한번 가볍게 훑어보세요.
    - **(예상 소요시간)**: 1시간

- [x] **필수 라이브러리(패키지) 추가** ✅ 2025-09-20
    - **(힌트)**: `pubspec.yaml` 파일을 열고, `dependencies:` 섹션에 `sqflite`, `path_provider`, `riverpod`, `go_router` 패키지를 추가하세요. `flutter pub get` 명령어로 설치하는 것을 잊지 마세요.
    - **(예상 소요시간)**: 2시간

- [x] **데이터베이스 헬퍼(Helper) 클래스 작성** ✅ 2025-09-20
    - **(힌트)**: `sqflite` 사용법을 검색해보세요. 데이터베이스를 열고, 기획서에 정의된 `cards`, `decks` 테이블을 생성하는 초기화 코드를 작성합니다. 처음에는 실제 데이터를 다루지 않고, 테이블을 만드는 것까지만 목표로 합니다.
    - **(예상 소요시간)**: 8시간

- [x] **기본 화면 라우팅(Routing) 설정** ✅ 2025-09-20
    - **(힌트)**: `go_router`를 사용해서 앱 실행 시 보여줄 첫 화면(초기에는 비어있는 화면)과 카드 리스트 화면을 연결하는 기본 경로를 설정합니다. 각 화면은 간단한 텍스트만 있는 임시 페이지로 만들어도 좋습니다.
    - **(예상 소요시간)**: 3시간

## **Sprint 2 (2주차): 빠른 메모 및 카드 생명주기 구현**

> **이번 주 목표**: 사용자가 실제로 메모를 쓰고 저장하는 핵심 기능을 완성합니다. 앱을 껐다 켜도 내가 쓴 내용이 남아있게 됩니다.

- [ ] **'새 카드 작성' 화면 UI 구현**
    - **(힌트)**: 앱의 첫 화면입니다. `Scaffold` 위젯 안에 제목을 입력할 `TextField`와 내용을 입력할 `TextField`(여러 줄 입력이 가능하도록 `maxLines` 속성 설정)를 배치하세요.
    - **(예상 소요시간)**: 4시간

- [ ] **카드 정보 모델(Model) 클래스 작성**
    - **(힌트)**: `cards` 테이블의 각 컬럼(id, title, content 등)에 해당하는 변수를 가진 `Card` 클래스를 만듭니다. `toMap()`, `fromMap()` 메소드를 만들어두면 DB와 데이터를 주고받을 때 편리합니다.
    - **(예상 소요시간)**: 3시간

- [ ] **카드 저장 기능 구현**
    - **(힌트)**: '저장' 버튼을 누르면, `TextField`에 입력된 내용을 `Card` 모델 객체에 담고, 지난주에 만든 DB 헬퍼를 사용해 `cards` 테이블에 `INSERT`하는 로직을 구현합니다.
    - **(예상 소요시간)**: 5시간

- [ ] **'카드 리스트' 화면 UI 및 기능 구현**
    - **(힌트)**: `ListView.builder` 위젯을 사용하면 효율적으로 리스트를 만들 수 있습니다. DB 헬퍼로 `cards` 테이블의 모든 데이터를 `SELECT`하여 가져온 뒤, 각 카드의 제목과 내용 일부를 화면에 표시합니다.
    - **(예상 소요시간)**: 6시간

- [ ] **카드 삭제 기능 구현**
    - **(힌트)**: 리스트의 각 항목을 길게 누르거나(Long Press), 스와이프(Swipe) 했을 때 삭제 확인 대화상자(AlertDialog)를 띄웁니다. '삭제'를 누르면 DB에서 해당 카드를 `DELETE`하고, 화면 리스트를 새로고침합니다.
    - **(예상 소요시간)**: 4시간

---

## **Sprint 3 (3주차): 리치 텍스트 및 기본 커스터마이징**

> **이번 주 목표**: 단순 텍스트를 넘어, 글자를 꾸미고 이미지를 첨부하여 카드를 더 풍부하게 만듭니다.

- [ ] **리치 텍스트 에디터 도입**
    - **(힌트)**: 직접 만드는 것은 매우 어렵습니다. `flutter_quill` 같은 검증된 리치 텍스트 에디터 패키지를 사용하는 것을 강력히 추천합니다. 패키지 문서를 따라 기존의 `TextField`를 대체해보세요.
    - **(예상 소요시간)**: 8시간

- [ ] **리치 텍스트 저장 및 표시 기능**
    - **(힌트)**: 리치 텍스트 에디터는 보통 JSON 형태의 데이터로 내용을 출력합니다. 이 JSON 문자열을 그대로 DB의 `content` 컬럼에 저장하고, 읽기 모드에서는 이 JSON 데이터를 다시 에디터에 넣어 보여주기만 하면(읽기 전용 모드로) 서식이 유지됩니다.
    - **(예상 소요시간)**: 4시간

- [ ] **카드 이미지 첨부 기능 구현**
    - **(힌트)**: `image_picker` 패키지를 사용하면 쉽게 갤러리나 카메라에 접근할 수 있습니다. 이미지를 선택하면 앱 내부 저장 공간에 복사하고, 그 파일 경로를 DB의 `front_image` 컬럼에 저장합니다.
    - **(예상 소요시간)**: 6시간

- [ ] **카드에 이미지 표시하기**
    - **(힌트)**: 읽기 모드에서 DB에 저장된 이미지 파일 경로를 읽어와 `Image.file()` 위젯으로 화면에 보여줍니다. 이미지가 없을 경우를 대비한 처리도 필요합니다.
    - **(예상 소요시간)**: 4시간

---

## **Sprint 4 (4주차): 검색 기능 및 MVP 안정화**

> **이번 주 목표**: 카드가 많아져도 쉽게 찾을 수 있도록 검색 기능을 추가하고, 지금까지 만든 기능들을 다듬어 첫 번째 버전을 완성합니다.

- [ ] **키워드 검색 기능 구현**
    - **(힌트)**: 카드 리스트 화면에 검색용 `TextField`를 추가합니다. 텍스트가 입력될 때마다 DB에 `SELECT ... WHERE title LIKE '%검색어%' OR content LIKE '%검색어%'` 와 같은 SQL 쿼리를 보내 결과를 필터링하고, 화면을 새로고침합니다.
    - **(예상 소요시간)**: 8시간

- [ ] **UI/UX 다듬기**
    - **(힌트)**: 앱의 전체적인 디자인 테마(색상, 폰트)를 적용하고, 버튼이나 카드 사이의 간격(Padding, Margin)을 조절하여 보기 좋게 만듭니다. 로딩 중임을 알려주는 인디케이터(CircularProgressIndicator)도 추가해보세요.
    - **(예상 소요시간)**: 6시간

- [ ] **1단계 기능 통합 테스트**
    - **(힌트)**: 직접 사용자가 되어, 앱을 처음 설치하는 것부터 카드를 10개 이상 만들고, 꾸미고, 검색하고, 삭제하는 모든 과정을 테스트합니다. 불편한 점이나 오류를 기록하세요.
    - **(예상 소요시간)**: 4시간

- [ ] **버그 수정 및 안정화**
    - **(힌트)**: 통합 테스트 과정에서 발견된 오류들을 수정합니다. 특히 앱이 예기치 않게 종료되는(Crash) 현상이 없는지 집중적으로 확인합니다.
    - **(예상 소요시간)**: 4시간
