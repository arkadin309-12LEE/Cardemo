# Sprint 4 (4주차): 검색 기능 및 Phase 1 안정화

> **이번 주 목표**: 카드가 많아져도 쉽게 찾을 수 있도록 검색 기능을 추가하고, 지금까지 만든 기능들을 다듬어 외부 피드백을 받을 수 있는 첫 번째 안정 버전을 완성합니다.
> 
> **사용자 테스트 목표**: 내가 작성했던 카드를 키워드로 검색해서 찾을 수 있다. 앱이 부드럽게 동작하며, 다른 사람에게 테스트를 부탁할 수 있는 수준이 된다.

---

## 상세 작업 목록

- [ ] **키워드 검색 기능 구현** (총 8시간)
    1. [ ] **(UI)**: 카드 리스트 화면의 `AppBar`에 검색 `IconButton`을 추가합니다. 이 버튼을 누르면 `TextField`가 나타나는 검색 UI로 전환됩니다.
    2. [ ] **(DB)**: `DatabaseHelper`에 `searchCards(String keyword)` 메소드를 추가합니다. 이 메소드는 `WHERE title LIKE ? OR content LIKE ?` SQL 구문을 사용하여 검색을 수행합니다.
    3. [ ] **(상태관리)**: `Riverpod`에 검색어 상태를 관리하는 `StateProvider`를 만듭니다.
    4. [ ] **(로직)**: 검색용 `TextField`의 내용이 변경될 때마다(`onChanged`), 검색어 `Provider`의 상태를 업데이트합니다.
    5. [ ] **(로직)**: 카드 목록을 보여주는 `Provider`가 검색어 `Provider`를 지켜보게 하고, 검색어가 있으면 `searchCards`를, 없으면 `getAllCards`를 호출하도록 로직을 수정합니다.

- [ ] **UI/UX 다듬기** (총 6시간)
    1. [ ] **(테마)**: `main.dart`의 `MaterialApp`에 `ThemeData`를 설정하여 앱의 전체적인 색상(primaryColor, accentColor 등)과 폰트 테마를 통일합니다.
    2. [ ] **(간격 조정)**: 앱의 모든 화면을 다시 한번 검토하며, `Padding`, `Margin`, `SizedBox` 등을 사용하여 위젯 간의 간격을 일관성 있게 조정합니다.
    3. [ ] **(로딩 인디케이터)**: DB에서 데이터를 불러오는 동안처럼, 시간이 걸릴 수 있는 작업 중에 `CircularProgressIndicator`를 보여주어 사용자가 기다리고 있음을 인지하게 합니다.
    4. [ ] **(빈 화면 처리)**: 카드 리스트가 비어있을 때 "첫 카드를 작성해보세요!"와 같은 안내 메시지와 아이콘을 보여주는 UI를 추가합니다.

- [ ] **1단계 기능 통합 테스트** (총 4시간)
    1. [ ] **(시나리오 작성)**: 사용자가 앱을 처음 실행해서부터 카드를 만들고, 글자를 꾸미고, 이미지를 넣고, 리스트에서 확인하고, 검색하고, 삭제하기까지의 모든 핵심 시나리오를 목록으로 작성합니다.
    2. [ ] **(테스트 수행)**: 실제 기기에서 위 시나리오를 그대로 따라하며, 각 단계에서 UI가 깨지거나, 버튼이 동작하지 않거나, 앱이 멈추는 등의 문제가 없는지 꼼꼼히 확인하고 기록합니다.
    3. [ ] **(엣지 케이스 테스트)**: 일부러 아주 긴 글을 입력하거나, 아주 큰 이미지를 첨부하거나, 검색어 없이 검색 버튼을 누르는 등 예외적인 상황도 테스트해봅니다.

- [ ] **버그 수정 및 안정화** (총 4시간)
    1. [ ] **(이슈 목록화)**: 위 테스트 과정에서 발견된 모든 문제점들을 우선순위에 따라 목록으로 만듭니다. (예: 앱 강제 종료 > 기능 미작동 > UI 깨짐)
    2. [ ] **(디버깅)**: 우선순위가 높은 이슈부터 원인을 분석합니다. `print()` 문이나 Flutter DevTools의 디버거를 사용하여 문제의 원인을 찾아냅니다.
    3. [ ] **(코드 수정)**: 원인이 파악된 버그를 수정합니다.
    4. [ ] **(회귀 테스트)**: 버그를 수정한 후, 해당 기능뿐만 아니라 관련된 다른 기능들도 여전히 잘 동작하는지 다시 한번 테스트합니다.

- [ ] **(Phase 1) 성능 점검** (총 2시간)
    1. [ ] **(측정)**: Flutter DevTools를 사용하여 앱 시작 속도, 카드 리스트 스크롤 시의 프레임 속도(FPS)를 측정하고 기록합니다.
    2. [ ] **(개선)**: 성능 저하가 발견된 부분이 있다면, 간단한 최적화 작업을 진행합니다. (예: 불필요한 위젯 재생성 방지)

- [ ] **(Phase 1) 외부 피드백용 MVP 빌드 생성** (총 2시간)
    1. [ ] **(빌드)**: `flutter build apk --release` 또는 `flutter build ipa --release` 명령어로 외부 테스터에게 전달할 앱 설치 파일을 생성합니다.
    2. [ ] **(문서화)**: 테스터에게 앱 설치 방법과 중점적으로 테스트해줬으면 하는 기능 목록을 간단히 안내하는 문서를 작성합니다.
