# Sprint 14-18 (5주): 최종 안정화 및 배포 준비

> **이번 주 목표**: 앱의 모든 기능을 최종적으로 점검하고, 사용자들이 다운로드할 수 있도록 구글 플레이 스토어와 애플 앱스토어에 출시할 준비를 완료합니다.
> 
> **사용자 테스트 목표**: 정식 출시 버전과 동일한 수준의 앱을 사용하며, 불편함이나 버그가 없는지 최종적으로 확인한다.

---

## 상세 작업 목록

- [ ] **(Phase 3) 최종 통합 테스트 및 피드백 반영** (약 20시간)
    1. [ ] **(내부 테스트)**: 개발팀 내부에서 모든 기능이 기획서대로 동작하는지, 빠진 기능은 없는지 최종적으로 검수합니다.
    2. [ ] **(외부 베타 테스트)**: Phase 1, 2에 참여했던 외부 테스터 그룹에게 최종 후보 버전을 배포하여, 마지막 피드백을 수집합니다.
    3. [ ] **(피드백 반영)**: 수집된 피드백 중, 정식 출시 전에 반드시 수정해야 할 치명적인 버그나 사용성 문제를 해결합니다.
    4. [ ] **(회귀 테스트)**: 버그 수정 후, 기존 기능들이 여전히 잘 동작하는지 전체적으로 다시 한번 확인합니다.

- [ ] **(Phase 3) 최종 성능 최적화** (약 10시간)
    1. [ ] **(종합 측정)**: Flutter DevTools를 사용하여 앱 시작 속도, 화면 전환 속도, 메모리 사용량, 배터리 소모량 등 앱의 전반적인 성능을 종합적으로 측정합니다.
    2. [ ] **(최종 개선)**: 측정된 데이터를 기반으로, 사용자 경험에 영향을 미치는 성능 저하 요소를 마지막으로 개선하고 최적화합니다.
    3. [ ] **(저사양 기기 테스트)**: 가능하다면, 비교적 사양이 낮은 실제 기기에서 앱이 최소한의 사용성을 보장하는지 테스트합니다.

- [ ] **앱 아이콘 및 스플래시 스크린 적용** (약 5시간)
    1. [ ] **(에셋 준비)**: 1024x1024 픽셀 이상의 고해상도 앱 아이콘 이미지를 준비합니다.
    2. [ ] **(패키지 설정)**: `flutter_launcher_icons` 패키지 설정을 통해 안드로이드/iOS 앱 아이콘을 자동 생성합니다.
    3. [ ] **(스플래시 적용)**: `flutter_native_splash` 패키지를 사용하여 앱 로딩 중에 표시될 스플래시 화면을 설정합니다.

- [ ] **앱스토어 제출용 정보 및 에셋 준비** (약 8시간)
    1. [ ] **(스크린샷 제작)**: 다양한 기기 사이즈(스마트폰, 태블릿)에서 앱의 주요 기능 화면을 캡처하여 스토어 게시용 스크린샷을 만듭니다.
    2. [ ] **(문서 작성)**: 앱 이름, 부제, 설명, 개인정보처리방침 URL, 지원 URL 등 스토어에 필요한 모든 텍스트 정보를 작성하고 검토합니다.
    3. [ ] **(정책 확인)**: Google Play와 Apple App Store의 최신 정책 가이드라인을 다시 한번 확인하여 위반 사항이 없는지 점검합니다.

- [ ] **최종 배포 빌드 생성 및 검수** (약 7시간)
    1. [ ] **(코드 서명 설정)**: 안드로이드의 Keystore와 iOS의 Signing Certificate 설정을 완료하여, 앱을 정식으로 서명할 준비를 합니다.
    2. [ ] **(빌드 실행)**: `flutter build appbundle --release` 와 `flutter build ipa --release` 명령어로 각 스토어에 제출할 최종 릴리즈 파일을 생성합니다.
    3. [ ] **(최종 검수)**: 생성된 파일을 실제 기기에 설치하여, 개발 버전과 동일하게 모든 기능이 완벽히 동작하는지 마지막으로 검수합니다.
