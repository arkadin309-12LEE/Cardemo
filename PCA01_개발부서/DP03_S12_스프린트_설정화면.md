# Sprint 12 (12주차): 설정 - 화면 및 테마

> **이번 주 목표**: 사용자가 자신의 취향에 맞게 앱의 외관을 설정할 수 있는 기능을 제공합니다.
> 
> **사용자 테스트 목표**: 앱 전체를 어두운 테마로 바꾸거나, 글자 크기를 더 크게 키워서 편하게 볼 수 있다.

---

## 상세 작업 목록

- [ ] **설정 화면 레이아웃 구현** (총 5시간)
    1. [ ] **(파일 생성)**: `settings_screen.dart` 파일을 생성하고, `go_router`에 `/settings` 경로를 추가합니다.
    2. [ ] **(UI)**: `Scaffold`와 `ListView`를 사용하여 설정 메뉴의 기본 구조를 만듭니다.
    3. [ ] **(UI)**: ‘화면/테마 설정’, ‘데이터 관리’, ‘정보’ 등 각 섹션의 제목과 구분선을 추가하여 가독성을 높입니다.

- [ ] **라이트/다크 모드 전환 기능** (총 8시간)
    1. [ ] **(상태관리)**: 앱의 현재 테마 모드(`ThemeMode.light` 또는 `ThemeMode.dark`)를 관리하는 `StateNotifierProvider` (Riverpod)를 작성합니다.
    2. [ ] **(UI)**: 설정 화면에 `SwitchListTile` 위젯을 추가하여, 스위치를 켜고 끌 때 위 `Provider`의 상태를 변경하도록 구현합니다.
    3. [ ] **(테마 정의)**: `MaterialApp.router`에 `theme` 속성에는 밝은 테마의 `ThemeData`를, `darkTheme` 속성에는 어두운 테마의 `ThemeData`를 정의합니다.
    4. [ ] **(앱 적용)**: `MaterialApp.router`의 `themeMode` 속성이 위에서 만든 `Provider`의 상태를 바라보게 하여, 상태 변경 시 앱 전체의 테마가 실시간으로 바뀌도록 연결합니다.

- [ ] **폰트 크기 및 종류 변경 기능** (총 7시간)
    1. [ ] **(UI)**: 설정 화면에 ‘글자 크기’ 항목을 추가하고, 슬라이더(`Slider`)나 드롭다운 메뉴(`DropdownButton`)로 크기를 조절하는 UI를 구현합니다.
    2. [ ] **(상태관리)**: 선택된 글자 크기 배율(예: 1.0, 1.2)을 관리하는 `StateProvider`를 만듭니다.
    3. [ ] **(저장소 연동)**: `shared_preferences`를 사용하여, 사용자가 글자 크기를 변경할 때마다 그 값을 기기에 저장합니다. 앱 실행 시에는 저장된 값을 읽어와 `Provider`의 초기값으로 설정합니다.
    4. [ ] **(앱 적용)**: `MaterialApp`의 `builder` 속성을 사용하여, 앱의 최상위에서 `MediaQuery`를 재정의하고 `textScaleFactor` 값을 위 `Provider`의 상태와 연결하여 앱 전체의 폰트 크기를 동적으로 변경합니다.
