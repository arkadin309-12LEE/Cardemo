# Sprint 7 (7주차): 덱 커스터마이징 (이미지)

> **이번 주 목표**: 밋밋한 덱 목록에 생기를 불어넣습니다. 각 덱을 대표하는 이미지를 설정할 수 있게 합니다.
> 
> **사용자 테스트 목표**: ‘여행’ 덱의 커버를 멋진 풍경 사진으로 바꿀 수 있다.

---

## 상세 작업 목록

- [ ] **덱 커버 이미지 설정 기능** (총 6시간)
    1. [ ] **(UI)**: 덱 정보 수정 화면 또는 덱 보기 화면의 `AppBar`에 ‘커버 이미지 변경’ 아이콘 버튼을 추가합니다.
    2. [ ] **(패키지 활용)**: `image_picker` 패키지를 사용하여 갤러리에서 이미지를 선택하는 기능을 구현합니다.
    3. [ ] **(파일 처리)**: 선택된 이미지를 앱 내부 디렉토리에 고유한 이름으로 복사하고, 그 경로를 `decks` 테이블의 `cover_image` 컬럼에 `UPDATE` 합니다.

- [ ] **덱 리스트에 커버 이미지 표시** (총 6시간)
    1. [ ] **(UI 수정)**: 덱 리스트의 각 아이템 UI를 `Card` 위젯이나 `Container`로 감싸고, `decoration` 속성을 사용하여 배경 이미지를 표시할 수 있도록 구조를 변경합니다.
    2. [ ] **(로직)**: 각 덱의 `cover_image` 경로가 있는지 확인합니다.
    3. [ ] **(UI 표시)**: 경로가 있다면 `FileImage(File(imagePath))`를 `BoxDecoration`의 `image` 속성에 설정하여 이미지를 배경으로 표시합니다.
    4. [ ] **(예외 처리)**: 이미지가 없다면, 덱의 기본 색상 테마나 기본 아이콘이 표시되도록 합니다.

- [ ] **덱 보기 화면 상단에 커버 이미지 표시** (총 6시간)
    1. [ ] **(UI 구조 변경)**: 덱 내부 카드 리스트 화면의 `Scaffold`를 `CustomScrollView`와 `SliverAppBar`를 사용하는 구조로 변경하여, 스크롤 시 상단 영역이 자연스럽게 축소되는 효과를 준비합니다.
    2. [ ] **(UI)**: `SliverAppBar`의 `flexibleSpace` 영역에 덱 커버 이미지를 표시합니다.
    3. [ ] **(UX 개선)**: 스크롤을 올리면 이미지가 커지고, 내리면 작아지면서 제목만 남는 동적인 UI를 구현하여 시각적 만족도를 높입니다.
