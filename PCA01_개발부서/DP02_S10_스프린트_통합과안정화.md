# Sprint 10 (10주차): 2단계 통합 및 안정화

> **이번 주 목표**: 지금까지 만든 모든 카드와 덱 관련 기능들이 서로 문제없이 잘 동작하는지 종합적으로 테스트하고, 2단계 완성을 위한 외부 피드백을 준비합니다.
> 
> **사용자 테스트 목표**: 카드와 덱의 모든 기능이 서로 충돌 없이 부드럽게 동작한다. 새로운 덱 기능을 다른 사람에게 테스트해달라고 요청할 수 있다.

---

## 상세 작업 목록

- [ ] **Phase 1, 2 기능 통합 테스트** (총 8시간)
    1. [ ] **(시나리오 테스트)**: 새 카드를 만들고, 서식을 입히고, 이미지를 넣은 뒤, 새 덱을 만들어 그 안에 카드를 추가합니다.
    2. [ ] **(시나리오 테스트)**: 방금 만든 덱의 커버 이미지와 타이틀 위치/크기/각도를 변경하고 저장합니다.
    3. [ ] **(시나리오 테스트)**: 덱 리스트와 카드 리스트를 오가며 정렬 순서를 바꿔보고, 특정 키워드로 ‘덱 포함’ 검색을 수행합니다.
    4. [ ] **(시나리오 테스트)**: 덱을 삭제했을 때, 내부의 카드들이 낱장 카드로 잘 이동되는지 확인합니다.

- [ ] **테스트 코드 보강** (총 6시간)
    1. [ ] **(단위 테스트)**: `DatabaseHelper`에 추가된 덱 관련 메소드들(`insertDeck`, `getCardsInDeck` 등)에 대한 단위 테스트를 작성합니다.
    2. [ ] **(단위 테스트)**: 덱에 카드를 추가하고 제외하는 로직의 정확성을 검증하는 테스트를 추가합니다.

- [ ] **2단계 버그 수정 및 안정화** (총 8시간)
    1. [ ] **(이슈 해결)**: 통합 테스트 과정에서 발견된 모든 버그와 UI 어색한 부분들을 목록으로 만들고, 우선순위가 높은 것부터 해결합니다.
    2. [ ] **(성능 검토)**: 덱에 카드가 100개 이상 있을 경우를 가정하여, 리스트 스크롤 성능이나 검색 속도가 느려지지 않는지 확인하고 개선합니다.
    3. [ ] **(코드 리뷰)**: 동료 개발자가 있다면, Phase 2에서 작성된 코드 전체에 대해 코드 리뷰를 진행하여 잠재적인 문제를 찾아냅니다.

- [ ] **(Phase 2) 성능 점검** (총 2시간)
    1. [ ] **(측정)**: 덱에 카드를 100개 이상 넣은 상태에서 덱 리스트, 카드 리스트의 스크롤 성능(FPS)을 집중적으로 측정합니다.
    2. [ ] **(개선)**: 덱 타이틀 커스터마이징, 덱 커버 이미지 로딩 등 Phase 2에서 추가된 기능 중 성능 저하를 유발하는 부분을 찾아 개선합니다.

- [ ] **(Phase 2) 외부 피드백용 빌드 생성** (총 2시간)
    1. [ ] **(빌드)**: 덱 기능이 포함된 새로운 버전의 앱 설치 파일을 생성합니다.
    2. [ ] **(문서화)**: Phase 1 테스터에게 변경된 기능(특히 덱 관련)과 새롭게 테스트가 필요한 부분을 안내하는 문서를 작성합니다.
