# 카데모(CARDEMO) Phase 3 상세 작업계획서 (초심자용)

## 1. Phase 3 목표

이 단계의 목표는 **앱의 완성도를 높이고 정식 배포를 준비하는 것**입니다. ‘랜덤 카드 뽑기’와 같은 핵심 재미 요소를 추가하고, 데이터 관리 기능을 통해 앱의 안정성을 확보하며, 사용자가 앱을 다운로드할 수 있도록 앱스토어에 출시할 준비를 모두 마칩니다.

---

## **Sprint 11 (11주차): 랜덤 뽑기 및 애니메이션**

> **이번 주 목표**: 이 앱의 가장 큰 재미 요소인 ‘랜덤 카드 뽑기’ 기능을 화려한 애니메이션과 함께 구현합니다.

- [ ] **카드 섞기 및 랜덤 추출 로직 구현**
    - **(힌트)**: 덱 보기 화면에 ‘랜덤 뽑기’ 버튼을 추가합니다. 버튼을 누르면, 덱에 포함된 카드 리스트를 `List.shuffle()` 메소드로 섞은 뒤, 첫 번째 아이템을 뽑는 간단한 로직으로 구현할 수 있습니다.
    - **(예상 소요시간)**: 6시간

- [ ] **카드 뒤집기 애니메이션 구현**
    - **(힌트)**: 이 기능은 앱의 인상을 좌우합니다. `AnimationController`와 `Transform` 위젯을 함께 사용하는 방법을 공부해보세요. `Transform.rotateY()`를 이용하면 Y축을 기준으로 회전하는 3D 뒤집기 효과를 만들 수 있습니다.
    - **(예상 소요시간)**: 10시간

- [ ] **랜덤 뽑기 화면 UI/UX 구현**
    - **(힌트)**: 카드를 뽑으면, 화면이 어두워지면서 뽑힌 카드가 뒷면으로 중앙에 나타나는 특별한 화면을 구성합니다. 사용자가 카드를 터치하면, 위에서 만든 뒤집기 애니메이션이 실행되도록 연결합니다.
    - **(예상 소요시간)**: 6시간

---

## **Sprint 12 (12주차): 설정 - 화면 및 테마**

> **이번 주 목표**: 사용자가 자신의 취향에 맞게 앱의 외관을 설정할 수 있는 기능을 제공합니다.

- [ ] **설정 화면 레이아웃 구현**
    - **(힌트)**: 하단 탭 메뉴와 연결될 설정 화면의 기본 UI를 구성합니다. ‘화면/테마 설정’, ‘데이터 관리’ 등 각 섹션을 구분하여 메뉴 리스트를 만듭니다.
    - **(예상 소요시간)**: 5시간

- [ ] **라이트/다크 모드 전환 기능**
    - **(힌트)**: Flutter의 `ThemeData`를 사용하는 방법을 공부하세요. `Riverpod` 같은 상태 관리 도구를 사용하여 앱 전체의 테마 모드(밝게/어둡게) 상태를 관리하고, 사용자가 스위치로 이 상태를 변경할 수 있게 합니다.
    - **(예상 소요시간)**: 8시간

- [ ] **폰트 크기 및 종류 변경 기능**
    - **(힌트)**: `shared_preferences` 패키지를 사용하여 사용자가 선택한 폰트 크기 값을 저장합니다. 앱이 시작될 때 이 값을 읽어와 앱 전체의 기본 텍스트 스타일에 적용합니다.
    - **(예상 소요시간)**: 7시간

---

## **Sprint 13 (13주차): 설정 - 데이터 관리**

> **이번 주 목표**: 사용자의 소중한 데이터가 사라지지 않도록, 데이터를 외부로 저장(백업)하고 다시 불러올(복원) 수 있는 기능을 구현합니다.

- [ ] **데이터 백업(내보내기) 기능**
    - **(힌트)**: DB의 모든 카드와 덱 정보를 `List<Map<String, dynamic>>` 형태로 읽어온 뒤, `jsonEncode` 함수를 사용해 하나의 JSON 문자열로 만듭니다. 이 문자열을 파일로 저장하고, `share_plus` 패키지를 사용해 사용자가 원하는 곳(이메일, 클라우드 드라이브 등)으로 공유할 수 있게 합니다.
    - **(예상 소요시간)**: 10시간

- [ ] **데이터 복원(가져오기) 기능**
    - **(힌트)**: 사용자가 백업 파일을 선택하면, 그 파일의 JSON 문자열을 `jsonDecode`로 다시 파싱합니다. 기존 DB 데이터를 모두 삭제한 후, 파싱된 데이터를 카드와 덱 객체로 변환하여 DB에 다시 `INSERT` 합니다. (데이터를 덮어쓴다는 경고 메시지를 반드시 보여줘야 합니다.)
    - **(예상 소요시간)**: 8시간

- [ ] **데이터 초기화 기능**
    - **(힌트)**: 설정 화면에 ‘모든 데이터 삭제’ 버튼을 만듭니다. 사용자가 버튼을 누르면, 정말로 삭제할 것인지 재차 확인하는 대화상자(AlertDialog)를 보여줍니다. 확인 시, DB의 모든 테이블에서 데이터를 `DELETE` 합니다.
    - **(예상 소요시간)**: 4시간

---

## **Sprint 14-18 (5주차): 최종 안정화 및 배포 준비**

> **이번 주 목표**: 앱의 모든 기능을 최종적으로 점검하고, 사용자들이 다운로드할 수 있도록 구글 플레이 스토어와 애플 앱스토어에 출시할 준비를 완료합니다.

- [ ] **최종 통합 테스트 및 버그 수정**
    - **(힌트)**: 가족이나 친구 등 제3자에게 앱을 직접 사용해보게 하고 피드백을 받으세요. 내가 미처 생각하지 못했던 사용 패턴에서 버그가 발견될 수 있습니다. 발견된 모든 버그를 수정합니다.
    - **(예상 소요시간)**: 20시간

- [ ] **성능 최적화**
    - **(힌트)**: Flutter DevTools의 ‘Performance’ 탭을 사용하여 앱의 프레임 속도(FPS)를 확인하고, 60fps 이하로 떨어지는 구간이 없는지 확인하고 최적화합니다. 특히 이미지나 리스트가 많은 화면을 집중적으로 점검합니다.
    - **(예상 소요시간)**: 10시간

- [ ] **앱 아이콘 및 스플래시 스크린 적용**
    - **(힌트)**: `flutter_launcher_icons`와 `flutter_native_splash` 패키지를 사용하면 앱 아이콘과 앱 실행 시 잠시 나타나는 스플래시 화면을 쉽게 설정할 수 있습니다.
    - **(예상 소요시간)**: 5시간

- [ ] **앱스토어 제출용 정보 및 에셋 준비**
    - **(힌트)**: 스토어에 등록할 앱의 이름, 설명, 스크린샷(다양한 기기 사이즈), 개인정보처리방침 URL 등을 미리 문서로 정리해둡니다. 기획서의 ‘배포 계획’ 섹션을 참고하세요.
    - **(예상 소요시간)**: 8시간

- [ ] **최종 배포 빌드 생성 및 테스트**
    - **(힌트)**: `flutter build appbundle --release` (안드로이드), `flutter build ipa --release` (iOS) 명령어를 통해 실제 스토어에 올릴 최종 파일을 생성합니다. 이 파일을 실제 핸드폰에 설치하여 마지막으로 모든 기능이 정상 동작하는지 확인합니다.
    - **(예상 소요시간)**: 7시간
