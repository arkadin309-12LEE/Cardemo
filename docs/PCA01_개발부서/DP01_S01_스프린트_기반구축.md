# Sprint 1 (1주차): 프로젝트 기반 구축

> **이번 주 목표**: 코딩을 시작할 수 있는 깨끗한 도화지를 준비합니다. Flutter를 설치하고, 데이터베이스 파일을 만들 준비를 합니다.
> 
> **사용자 테스트 목표**: (빌드 설치 확인) 앱이 정상적으로 설치되고 실행되는지 확인.

---

## 상세 작업 목록

- [x] **Flutter 개발 환경 구축** (총 4시간) ✅ 2025-09-20
    1. [x] **(학습)**: Flutter 공식 홈페이지의 설치 가이드를 정독합니다. ✅ 2025-09-20
    2. [x] **(설치)**: 내 운영체제(Windows/Mac)에 맞는 Flutter SDK를 다운로드하고 압축을 해제합니다. ✅ 2025-09-20
    3. [x] **(환경설정)**: Flutter 명령어(CLI)를 어디서든 사용할 수 있도록 환경 변수(Path)를 설정합니다. ✅ 2025-09-20
    4. [x] **(검증)**: 터미널에서 `flutter doctor` 명령어를 실행하고, 모든 항목에 녹색 체크 표시가 나타나는지 확인합니다. (Android toolchain, VS Code 등) ✅ 2025-09-20

- [x] **신규 Flutter 프로젝트 생성** (총 1시간) ✅ 2025-09-20
    1. [x] **(실행)**: 터미널에서 내가 원하는 폴더로 이동한 뒤, `flutter create cardemo` 명령어를 실행합니다. ✅ 2025-09-20
    2. [x] **(탐색)**: 생성된 `cardemo` 폴더를 VS Code로 열고, `lib`, `pubspec.yaml`, `assets` 등 주요 폴더와 파일의 역할을 눈으로 익힙니다. ✅ 2025-09-20
    3. [x] **(테스트 실행)**: `lib/main.dart` 파일을 열고, F5 키를 눌러 에뮬레이터 또는 실제 기기에서 기본 카운터 앱이 실행되는지 확인합니다. ✅ 2025-09-20

- [x] **필수 라이브러리(패키지) 추가** (총 2시간) ✅ 2025-09-20
    1. [x] **(문서 확인)**: `pub.dev` 사이트에서 `sqflite`, `path_provider`, `flutter_riverpod`, `go_router` 패키지의 최신 버전을 확인합니다. ✅ 2025-09-20
    2. [x] **(설정)**: `pubspec.yaml` 파일을 열고 `dependencies:` 아래에 위 패키지들을 추가합니다. ✅ 2025-09-20
    3. [x] **(설치)**: 터미널에서 `flutter pub get` 명령어를 실행하거나, `pubspec.yaml` 파일을 저장하여 패키지가 다운로드되는 것을 확인합니다. ✅ 2025-09-20

- [x] **데이터베이스 헬퍼(Helper) 클래스 작성** (총 8시간) ✅ 2025-09-20
    1. [x] **(파일 생성)**: `lib` 폴더 아래에 `data` 또는 `database` 폴더를 만들고, `database_helper.dart` 파일을 생성합니다. ✅ 2025-09-20
    2. [x] **(기본 구조)**: `DatabaseHelper` 클래스를 만들고, 데이터베이스 인스턴스를 저장할 `_database` 변수를 선언합니다. (싱글턴 패턴을 적용하면 좋습니다.) ✅ 2025-09-20
    3. [x] **(초기화 함수)**: `initDB()` 같은 이름의 비동기(`async`) 함수를 만듭니다. 이 함수에서 `path_provider`로 DB 파일 경로를 얻고, `sqflite`의 `openDatabase`를 호출합니다. ✅ 2025-09-20
    4. [x] **(테이블 생성)**: `openDatabase`의 `onCreate` 콜백 함수 안에서, 기획서에 명시된 `cards`, `decks`, `settings` 테이블을 생성하는 `CREATE TABLE ...` SQL 쿼리를 실행하도록 작성합니다. ✅ 2025-09-20

- [x] **기본 화면 라우팅(Routing) 설정** (총 3시간) ✅ 2025-09-20
    1. [x] **(파일 생성)**: `lib` 폴더 아래에 `presentation` 또는 `ui` 폴더를, 그 안에 `screens` 폴더를 만듭니다. `new_card_screen.dart`, `card_list_screen.dart` 두 개의 임시 파일을 생성합니다. ✅ 2025-09-20
    2. [x] **(임시 화면 작성)**: 각 화면 파일에 `StatelessWidget`으로 간단한 `Scaffold`와 `Text` 위젯만 포함된 임시 화면을 만듭니다. (예: "새 카드 화면", "카드 리스트 화면") ✅ 2025-09-20
    3. [x] **(라우터 설정)**: `go_router` 사용법에 따라, 앱의 최상위 경로(`/`)는 `NewCardScreen`으로, `/list` 경로는 `CardListScreen`으로 연결하는 `GoRouter` 설정 코드를 작성합니다. ✅ 2025-09-20
    4. [x] **(앱에 적용)**: `main.dart` 파일의 `MaterialApp`을 `MaterialApp.router`로 변경하고, 위에서 만든 라우터 설정을 적용합니다. ✅ 2025-09-20
