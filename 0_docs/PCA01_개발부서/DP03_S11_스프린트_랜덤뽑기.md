# Sprint 11 (11주차): 랜덤 뽑기 및 애니메이션

> **이번 주 목표**: 이 앱의 가장 큰 재미 요소인 ‘랜덤 카드 뽑기’ 기능을 화려한 애니메이션과 함께 구현합니다.
> 
> **사용자 테스트 목표**: ‘명언’ 덱에서 오늘의 명언을 무작위로 한 장 뽑고, 직접 터치해서 뒤집어보는 재미를 느낄 수 있다.

---

## 상세 작업 목록

- [ ] **(사전 작업) 기술 검증 프로토타입 제작** (총 4시간)
    1. [ ] **(목표)**: 3D 카드 뒤집기 애니메이션의 성능과 자연스러움을 미리 검증합니다.
    2. [ ] **(구현)**: 별도의 테스트용 `dart` 파일에서, `AnimationController`와 `Transform.rotateY`를 사용하여 앞면과 뒷면이 뒤집히는 간단한 카드 위젯 예제를 만듭니다.
    3. [ ] **(검증)**: 저사양 기기나 에뮬레이터에서도 애니메이션이 부드럽게(60fps) 동작하는지 확인하고, 어색한 부분이 없는지 점검합니다.

- [ ] **카드 섞기 및 랜덤 추출 로직 구현** (총 6시간)
    1. [ ] **(UI)**: 덱 보기 화면의 하단 메뉴에 ‘랜덤 뽑기’ 아이콘 버튼을 추가합니다.
    2. [ ] **(로직)**: 버튼을 누르면, 현재 덱에 포함된 카드 리스트를 `List.from()`으로 복사한 뒤, `shuffle()` 메소드를 이용해 리스트를 무작위로 섞습니다.
    3. [ ] **(로직)**: 섞인 리스트의 첫 번째 카드(`shuffledList.first`)를 ‘뽑힌 카드’로 선택합니다.
    4. [ ] **(화면 이동)**: 뽑힌 카드 정보를 파라미터로 하여, 새로운 ‘카드 뽑기 결과’ 화면으로 이동시킵니다.

- [ ] **카드 뒤집기 애니메이션 구현** (총 10시간)
    - **(참고)**: 위의 프로토타입 제작 결과에 따라 예상 소요시간이 변경될 수 있습니다.
    1. [ ] **(학습)**: Flutter의 `AnimationController`와 `Transform.rotateY`를 함께 사용하는 방법을 학습합니다. 3D 회전 효과를 위해 `Transform`의 `alignment`를 `Alignment.center`로 설정하는 것이 중요합니다.
    2. [ ] **(위젯 생성)**: `StatefulWidget`으로 `FlippableCard` 같은 이름의 재사용 가능한 위젯을 만듭니다.
    3. [ ] **(애니메이션 설정)**: `AnimationController`를 초기화하고, 카드가 뒤집히는 각도(0 ~ PI)를 제어하는 `Animation<double>`을 생성합니다.
    4. [ ] **(UI 구현)**: `AnimatedBuilder`를 사용하여 애니메이션 값의 변화에 따라 `Transform.rotateY` 위젯을 다시 그리도록 구현합니다. 회전 각도가 90도(PI/2)를 넘어가는 시점에 앞면과 뒷면 위젯을 교체하여 뒤집히는 것처럼 보이게 만듭니다.

- [ ] **랜덤 뽑기 화면 UI/UX 구현** (총 6시간)
    1. [ ] **(파일 생성)**: `card_flip_screen.dart` 같은 이름의 새 화면 파일을 만듭니다.
    2. [ ] **(UI)**: 화면 전체를 어두운 반투명 레이어로 덮어(Modal) 집중도를 높입니다.
    3. [ ] **(UI)**: 화면 중앙에 위에서 만든 `FlippableCard` 위젯을 배치합니다. 처음에는 카드의 뒷면 이미지가 보이도록 설정합니다.
    4. [ ] **(로직)**: 사용자가 `FlippableCard`를 탭(onTap)하면, 애니메이션 컨트롤러의 `forward()` 메소드를 호출하여 뒤집기 애니메이션을 실행하도록 연결합니다.
