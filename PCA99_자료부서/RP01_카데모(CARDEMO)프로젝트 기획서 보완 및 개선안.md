# 카데모(CARDEMO) 프로젝트 기획서 보완 및 개선안

## 📋 현재 기획서 분석 결과

### ✅ 기존 기획서의 강점
1. **명확한 프로젝트 아이덴티티**: 카데모라는 브랜드명과 카드+덱+메모의 핵심 컨셉이 잘 정립됨
2. **구체적인 사용자 시나리오**: 로딩화면부터 상세한 UI 동작까지 세밀하게 기획됨
3. **독창적인 차별화 요소**: 랜덤 카드 뽑기, DECK 시스템 등 경쟁사 대비 차별점 명확

## 3. 기능 명세 (상세 보완)

>[!function] 카드 기능 상세 설명

### 3.1 카드 작성, 편집, 읽기 모드

#### 카드 작성 모드
- **편집 툴바 기능**
  - 헤더(H1~H3): 제목 및 부제목 설정
  - 텍스트 포맷팅: 볼드, 이탤릭, 취소선, 밑줄
  - 색상 옵션: 글자 색상, 하이라이트(배경색)
  - 리스트 기능: 체크리스트, 번호 목록, 불릿 목록
  - 글자 크기: 12px~18px 범위 조절

#### 카드 편집 모드 
- **편집 모드 진입**: 카드 터치 → 편집 버튼 선택
- **실시간 미리보기**: 편집 내용 즉시 반영
- **자동 저장**: 3초마다 자동 저장, 편집 완료 시 최종 저장
- **편집 취소**: 뒤로가기 시 변경사항 저장 여부 확인 팝업

#### 카드 읽기 모드
- **기본 표시**: 카드 제목, 내용, 작성일시, 덱 소속 정보
- **상호작용**: 스크롤, 확대/축소, 공유 기능
- **빠른 액션**: 길게 터치 시 편집/삭제/덱 이동 메뉴 표시

### 3.2 카드 크기, 방향, 이미지 설정

#### 카드 크기 설정
- **기본 크기**: 화면 너비의 90%, 높이 자동 조절
- **커스텀 크기**: 최소 200x150px ~ 최대 화면 크기
- **종횡비 옵션**: 1:1(정사각형), 4:3(가로), 3:4(세로), 16:9(와이드)
- **내용 오버플로우**: 높이 초과 시 새 카드 생성 옵션 제공

#### 카드 방향 지정
- **세로 모드**: 기본 설정, 일반적인 메모 형태
- **가로 모드**: 넓은 콘텐츠용, 표나 긴 문장에 적합
- **자동 회전**: 기기 회전에 따른 자동 레이아웃 조정

#### 카드 이미지 설정
- **앞면 이미지**: 카드 상단 또는 배경으로 설정 가능
- **뒷면 이미지**: 카드 뒤집기 효과 시 표시되는 이미지
- **이미지 소스**: 갤러리, 카메라, 기본 템플릿 선택
- **이미지 편집**: 크롭, 필터, 투명도 조절 기능

>[!deck] DECK 기능 상세

### 3.3 DECK 생성, 조직화, 이미지 및 명칭 위치 설정

#### DECK 생성
- **신규 DECK**: 이름, 설명, 색상 테마 설정
- **카드 추가**: 개별 선택 또는 다중 선택으로 DECK에 추가
- **DECK 복사**: 기존 DECK 구조 복사하여 새 DECK 생성
- **DECK 병합**: 두 개 이상의 DECK을 하나로 통합

#### DECK 조직화
- **계층 구조**: 상위 DECK 내 하위 DECK 생성 가능 (최대 3단계)
- **카드 순서**: 드래그&드롭으로 카드 순서 변경
- **카드 필터링**: 태그, 날짜, 크기별 카드 정렬
- **DECK 통계**: 포함된 카드 수, 최근 수정일, 총 용량 표시

#### DECK 이미지 및 명칭 위치 설정
- **DECK 커버 이미지**: 350x200px 권장 크기
- **명칭 위치 옵션**: 상단 중앙, 상단 좌측, 하단 중앙, 이미지 오버레이
- **텍스트 스타일**: 폰트, 크기, 색상, 그림자 효과 설정
- **DECK 아이콘**: 기본 아이콘 또는 커스텀 이미지 설정

>[!search] 검색 및 정렬 기능

### 3.4 리스트 및 정렬 기능

#### 카드 리스트 표시
- **그리드 뷰**: 2열 또는 3열 카드 격자 배치
- **리스트 뷰**: 세로 스크롤 방식 상세 목록
- **카드 뷰**: 한 번에 하나씩 전체 화면 표시
- **뷰 전환**: 우상단 버튼으로 뷰 모드 즉시 변경

#### 정렬 옵션
- **시간순**: 최근 작성순(기본), 오래된 순, 최근 수정순
- **제목순**: 가나다 순, ABC 순, 역순 정렬
- **크기순**: 카드 크기(면적) 기준 정렬
- **태그순**: 태그 알파벳 순, 태그 없는 카드는 별도 그룹

#### DECK 우선 표시
- **DECK 섹션**: 리스트 상단에 DECK 목록 표시
- **카드 섹션**: DECK 하단에 독립 카드 목록 표시
- **통합 뷰**: DECK과 카드를 시간순으로 통합 표시 옵션

### 3.5 검색 기능 (제목, 내용, 태그)

#### 검색 인터페이스
- **검색바 위치**: 상단 고정, 스크롤 시에도 접근 가능
- **실시간 검색**: 입력과 동시에 결과 필터링
- **검색 기록**: 최근 검색어 5개까지 저장
- **음성 검색**: 마이크 버튼으로 음성 입력 지원

#### 검색 범위 설정
- **기본 검색**: 독립 카드만 검색
- **DECK 포함 검색**: 토글 버튼으로 DECK 내 카드도 포함
- **전체 검색**: 모든 카드와 DECK 명칭까지 검색
- **DECK별 검색**: 특정 DECK 내에서만 검색

#### 고급 검색 옵션
- **필터 조합**: 날짜 범위 + 태그 + 키워드 조합 검색
- **정규식 검색**: 고급 사용자용 패턴 매칭 검색
- **제외 검색**: 특정 키워드를 포함하지 않는 카드 검색
- **유사 검색**: 오타나 유사한 단어도 포함하여 검색

### 3.6 기타 부가 기능

#### 카드 랜덤 추출
- **DECK 셔플**: 선택한 DECK의 모든 카드 섞기
- **1장 뽑기**: 랜덤하게 한 장 선택하여 뒷면으로 표시
- **뒤집기 애니메이션**: 터치 시 3D 회전 효과로 앞면 표시
- **연속 뽑기**: 뽑은 카드 제외하고 다음 카드 뽑기

#### 카드 공유 및 내보내기
- **이미지 내보내기**: 카드를 PNG/JPG 이미지로 저장
- **텍스트 내보내기**: 마크다운 형식으로 텍스트 추출
- **DECK 공유**: QR 코드로 DECK 전체 공유
- **클라우드 백업**: Google Drive, iCloud 연동 백업

## 4. UI/UX 설계 (상세 보완)

>[!design] 화면 구성 및 사용자 흐름

### 4.1 화면 구성 및 사용자 플로우

#### 주요 화면 흐름도
```
스플래시 화면 
    ↓
메인 화면 (카드 뷰)
    ↓
┌─ 카드 리스트 ←→ DECK 리스트
│       ↓               ↓
└─ 카드 편집      DECK 관리
    ↓               ↓
카드 설정        DECK 설정
    ↓               ↓
    └─ 공통 설정 화면 ─┘
```

#### 네비게이션 구조
- **탭 기반**: 하단 3개 탭 (카드, DECK, 설정)
- **슬라이드**: 좌우 스와이프로 탭 간 이동
- **백 버튼**: Android 표준 뒤로가기 동작
- **깊이 제한**: 최대 3단계 깊이로 제한하여 사용자 혼란 방지

### 4.2 주요 화면별 디자인 콘셉트

#### 디자인 테마
- **색상 팔레트**: 
  - 주색상: #2196F3 (파란색 계열)
  - 보조색상: #FFC107 (황색 계열)  
  - 배경색: #FAFAFA (연한 회색)
  - 텍스트: #212121 (진한 회색)
- **타이포그래피**: Roboto(Android), SF Pro(iOS) 시스템 폰트
- **그림자 효과**: Material Design 표준 elevation 적용
- **모서리**: 8px 라운드 처리로 부드러운 느낌

#### 카드 디자인 원칙
- **일관성**: 모든 카드에 동일한 패딩(16px) 적용
- **계층감**: 카드 간격 12px, 그림자로 depth 표현
- **터치 영역**: 최소 48px×48px 터치 영역 확보
- **피드백**: 터치 시 리플 효과 또는 색상 변화

### 4.3 상호작용 및 편의성 고려사항

#### 터치 제스처
- **탭**: 카드 선택, 버튼 실행
- **롱프레스**: 컨텍스트 메뉴 표시 (편집, 삭제, 이동)
- **스와이프**: 좌우 스와이프로 카드 삭제 또는 즐겨찾기
- **핀치**: 카드 크기 조절 (확대/축소)
- **더블탭**: 카드 전체화면 보기

#### 접근성 (Accessibility)
- **VoiceOver/TalkBack**: 모든 UI 요소에 접근성 라벨 적용
- **고대비 모드**: 시각 장애 사용자를 위한 고대비 테마
- **폰트 크기**: 시스템 폰트 크기 설정 반영
- **색상 독립성**: 색상만으로 정보 전달하지 않음

#### 성능 최적화
- **레이지 로딩**: 화면에 보이는 카드만 렌더링
- **이미지 압축**: 카드 이미지 자동 압축 및 캐싱
- **메모리 관리**: 사용하지 않는 카드 데이터 메모리에서 해제
- **부드러운 애니메이션**: 60fps 유지를 위한 최적화

## 5. 기술 스택 및 개발 환경 (상세 보완)

>[!tech] 개발 언어 및 프레임워크

### 5.1 프론트엔드 기술 스택
- **메인 프레임워크**: Flutter 3.24+
- **상태 관리**: Riverpod 2.0 (Provider 패턴)
- **라우팅**: GoRouter (페이지 네비게이션)
- **애니메이션**: Flutter 내장 Animation API
- **UI 컴포넌트**: Material Design 3.0 기반 커스텀 위젯

### 5.2 백엔드 기술 스택 (선택사항)
- **로컬 우선**: SQLite (로컬 데이터베이스)
- **백엔드 (향후)**: Rust + Actix-web
- **데이터베이스**: PostgreSQL (클라우드 확장 시)
- **API**: RESTful API + JWT 인증
- **파일 저장**: 로컬 저장 우선, 향후 AWS S3

### 5.3 데이터베이스 및 저장 방식

#### 로컬 데이터 구조
```sql
-- 카드 테이블
CREATE TABLE cards (
    id TEXT PRIMARY KEY,
    deck_id TEXT NULL,
    title TEXT,
    content TEXT NOT NULL,
    tags TEXT, -- JSON 배열
    front_image TEXT,
    back_image TEXT,
    width INTEGER DEFAULT 0,
    height INTEGER DEFAULT 0,
    orientation TEXT DEFAULT 'portrait',
    created_at INTEGER,
    updated_at INTEGER
);

-- 덱 테이블  
CREATE TABLE decks (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT,
    cover_image TEXT,
    title_position TEXT DEFAULT 'top_center',
    created_at INTEGER,
    updated_at INTEGER
);

-- 설정 테이블
CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT
);
```

#### 파일 저장 구조
```
/app_directory/
├── cards/          # 카드 이미지
├── decks/          # 덱 커버 이미지  
├── backup/         # 백업 파일
└── cache/          # 임시 파일
```

### 5.4 협업 및 버전 관리 도구
- **Git**: GitHub 저장소

- **CI/CD**: GitHub Actions

## 7. 규정 준수 및 법적 고려사항 (신규 추가)

### 7.1 개인정보보호 규정 준수

#### 카데모 앱의 개인정보 처리 현황
- **수집하는 정보**: 
  - 카드 내용 (사용자 직접 입력)
  - 이미지 파일 (사용자 직접 업로드)
  - 앱 사용 통계 (선택사항)
- **수집하지 않는 정보**:
  - 연락처, 위치정보, 기기 식별정보
  - 외부 계정 연동 정보

#### 개인정보처리방침 필수 항목
```markdown
1. 개인정보의 처리 목적
   - 메모 카드 작성 및 관리 서비스 제공

2. 처리하는 개인정보의 항목
   - 필수: 없음 (모든 정보는 사용자 직접 입력)
   - 선택: 앱 사용 통계 (익명화된 사용 패턴)

3. 개인정보의 처리 및 보유기간
   - 사용자 삭제 시까지 (무제한)
   - 로컬 저장으로 외부 전송 없음

4. 개인정보의 제3자 제공
   - 제3자 제공 없음

5. 개인정보 처리의 위탁
   - 위탁 없음 (로컬 저장만 사용)
```

#### 앱 접근권한 정책
- **필수 권한**: 저장소 읽기/쓰기 (카드 이미지 저장용)
- **선택 권한**: 
  - 카메라 (사진 촬영 시에만 요청)
  - 갤러리 (이미지 선택 시에만 요청)
- **권한 거부 시**: 해당 기능만 제한, 다른 기능은 정상 사용 가능

### 7.2 Google Play Store 정책 준수

```yaml

데이터 공유: 없음  
보안 관행:
  - 전송 중 데이터 암호화: 해당없음 (로컬만 사용)
  - 저장된 데이터 암호화: 적용
  - 사용자 계정 삭제 요청: 지원 (로컬 데이터 삭제)
```

#### 콘텐츠 등급
- **연령 등급**: 전체 이용가 (3세+)
- **민감한 콘텐츠**: 없음
- **광고**: 없음
- **인앱 구매**: 없음 (무료 앱)

### 7.3 Apple App Store 심사 지침 준수
