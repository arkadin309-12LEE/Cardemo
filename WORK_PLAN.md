# CARDEMO 작업 계획서 (공동 운영용)

이 문서는 CARDEMO 프로젝트를 **사용자(PO) + Codex(개발 에이전트)**가 같은 기준으로 진행하기 위한 단일 기준 문서입니다.

---

## 1) 운영 원칙

- 이 문서를 스프린트 시작/종료 시점에 반드시 갱신합니다.
- 기능은 **1차(MVP) 우선**, 2차(확장)는 MVP 안정화 이후 진행합니다.
- 각 항목은 "완료 조건(Definition of Done)"을 충족해야 완료로 표시합니다.
- 우선순위 변경 시 변경 이력(Log)에 사유를 남깁니다.

---

## 2) 범위 정의

## 2.1 1차(MVP) 범위

- 카드 메모: 카드 생성/조회/수정/삭제(CRUD)
- 빠른 메모 진입 흐름(앱 시작 시 새 카드 작성 중심)
- 덱 기본 관리: 덱 생성/수정/삭제, 카드 덱에 추가/제외
- 기본 검색: 카드/덱 제목·내용·태그 기준 기본 검색
- 기본 정렬: 최신순/제목순 등 최소 정렬 옵션
- 로컬 저장소/DB 기본 스키마 + 마이그레이션 기초 구조

### 제외(2차로 이관)

- 카드 이미지 고급 편집(Crop/필터/투명도)
- 덱 타이틀 제스처 편집(드래그/핀치/회전)
- 3D 카드 뒤집기 애니메이션
- 고급 검색(정규식, 제외 검색 등)
- 고급 커스터마이징/공유/백업복원 고도화

## 2.2 2차(확장) 범위

- MVP 제외 기능 전체
- 커스터마이징/애니메이션/고급 검색/공유/백업복원
- 성능 최적화 및 UX 고도화

---

## 3) 역할 분담 (에이전트 팀 방식)

- **PO/기획(사용자)**: 우선순위 결정, 범위 승인, 수용 기준 최종 확정
- **PM 에이전트(Codex)**: 스프린트 계획, 일정/리스크/의존성 관리
- **개발 에이전트(Codex)**: 구현, 리팩터링, 문서화
- **QA 에이전트(Codex)**: 테스트 시나리오, 회귀 체크리스트, 릴리즈 체크

> 실제 운영은 한 에이전트가 역할을 전환하며 수행하고, 결과를 이 문서에 남깁니다.

---

## 4) MVP 백로그 (우선순위 순)

| ID | 작업 | 우선순위 | 상태 | 완료 조건(DoD) |
|---|---|---|---|---|
| M1 | 카드 데이터 모델 + CRUD | P0 | TODO | 생성/수정/삭제/조회가 정상 동작 |
| M2 | 빠른 메모 기본 화면/흐름 | P0 | TODO | 앱 시작 후 즉시 메모 작성 진입 가능 |
| M3 | 덱 모델 + CRUD | P0 | TODO | 덱 생성/수정/삭제/조회 가능 |
| M4 | 카드-덱 연결(추가/제외) | P0 | TODO | 카드의 덱 포함/제외가 정확히 반영 |
| M5 | 기본 검색(제목/내용/태그) | P1 | TODO | 키워드로 카드/덱 필터링 동작 |
| M6 | 기본 정렬(최신/제목) | P1 | TODO | 정렬 기준 전환 시 목록 반영 |
| M7 | 로컬 DB 버전 관리 기초 | P1 | TODO | 스키마 버전 확인/업데이트 루틴 동작 |
| M8 | 기본 테스트(단위/통합 최소셋) | P1 | TODO | 핵심 플로우 테스트 통과 |
| M9 | MVP 안정화/버그픽스 | P0 | TODO | P0/P1 버그 해소, 릴리즈 가능 상태 |

상태 값 규칙: `TODO` / `IN_PROGRESS` / `DONE` / `BLOCKED`

## 5) 스프린트 제안 (4주 MVP)

- **Week 1**: M1, M2
- **Week 2**: M3, M4
- **Week 3**: M5, M6, M7
- **Week 4**: M8, M9 + 내부 검증
